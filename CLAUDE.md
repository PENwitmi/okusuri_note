# CLAUDE.md - PharmaDex プロジェクト

## プロジェクト概要
**PharmaDex（ファーマデックス）- 未来薬局展示企画**

2025年夏の薬局チェーン向け未来型展示企画。薬学・医療の歴史から現在、そして未来への展望を、インタラクティブな展示で表現する大規模プロジェクト。

### コンセプト
「薬局の過去・現在・未来を一望できる体験型展示」
- 医薬品開発の歴史的マイルストーン
- 現代の課題（薬剤耐性、個別化医療など）
- 未来の薬局ビジョン（AI診断、遠隔医療、ナノ医療など）

## 特別ルール（重要）

### Git Worktree不使用の方針（2025-06-28）
**このプロジェクトではGit Worktreeを使用しません。**
- シンプルなWebサイト作成が目標のため、並列開発は不要

### CEO×Manager協働体制（2025-06-27強化）
**単なるメッセージ転送ではなく、深い思考による価値創造を**

#### なぜ協働思考が必要か
1. **表面的な指示では教育的価値が生まれない**
   - ❌「薬の使い分けを書いて」→ 添付文書の羅列
   - ✅「なぜこの薬が必要になったか」→ 本質的理解

2. **PharmaDexの核心価値を実現するため**
   - 「なぜ似た薬が複数存在するのか」への深い回答
   - 感動的なストーリーと実用的知識の最適な融合
   - 学生から臨床医まで各レベルでの価値創造

#### 協働思考の実践方法
- **薬効群モデル設計時**：
  - CEO「この薬群で最も重要な臨床的価値は？」
  - Manager「現場では○○で悩むので、△△を重点的に」
  - → 共に価値の方向性を定めてから開発者へ指示

- **ストーリー統合時**：
  - CEO「感動と実用性のバランスは？」
  - Manager「ストーリー入口から詳細への導線を○○に」
  - → 十分な検討後に具体的な構成を決定

運用方法：
- CEO：展示の大局的ビジョン、本質的な問いかけ
- Manager：技術的実現性、実践的価値の具体化
- 両者の深い対話により、AIの真の価値を発揮

#### 思考と実行のバランス（重要）
**⚠️ 避けるべき2つの極端**
1. **思考不足**：即座にタスク分割 → 表面的な成果物
2. **思考の沼**：延々と議論 → 成果物が生まれない

**✅ 理想的なワークフロー**
1. **協働思考フェーズ**（30-60分）
   - 本質的な価値を探る
   - 「なぜ」を明確化
   - 伝え方を設計

2. **実行判断基準**（3つのチェック）
   - □ 具体的な指示が書けるか
   - □ 成果物が明確か
   - □ 品質基準が定まったか
   → すべてYESで実行へ

3. **開発展開**
   - 明確な指示に変換
   - 創造性も確保
   - 納期設定と管理

**📊 実践的なタイムボックス**
```
0-30分：CEO×Manager協働思考
  - 薬効群の本質的価値を探る
  - 学習者のつまずきポイントを特定
  - ストーリーと知識の融合点を設計
  
30-45分：指示内容の具体化
  - 開発者への明確な成果物定義
  - 品質基準と評価ポイント設定
  - 参考資料とテンプレート準備
  
45-60分：開発者へのタスク配分
  - 各開発者の強みを活かした割当
  - 納期と中間チェックポイント設定
  - 質問を歓迎する雰囲気づくり
  
以降：進捗管理と品質確保
  - 定期的な進捗確認
  - 必要に応じた軌道修正
  - 成果物の統合と検証
```

**🎯 PharmaDexでの具体的な実践例**
- **ARBモデル作成時**：
  - 協働思考：「なぜロサルタンの後にカンデサルタンが必要だったか」
  - 実行判断：T/P比、半減期、臨床使い分けマトリックスが明確化
  - 開発展開：Dev1〜3に具体的な調査項目を配分
  
- **10大ストーリー作成時**：
  - 協働思考：「各ストーリーで最も伝えたい感動は何か」
  - 実行判断：感情曲線（希望→絶望→再生）の構成が確定
  - 開発展開：各開発者に4:3:3でストーリーを配分

### クロスリファレンス型Webサイト設計

#### 核心理念：「知識の網目が真の理解を生む」
薬学知識は相互依存的であり、一つの薬剤を理解するには歴史的文脈、薬理学的位置づけ、臨床的価値、試験的重要性のすべてが必要です。

#### 5つのコンテンツタイプ
1. **個別薬剤ページ**（約300ページ）
   - 各薬剤の詳細情報ハブ
   - 薬効群、ストーリー、国試情報への相互リンク

2. **薬効群解説ページ**（約30ページ）
   - 進化系統図と臨床使い分けマトリックス
   - なぜ似た薬が複数存在するのかへの回答

3. **ストーリーページ**（10大ストーリー＋α）
   - 感動から学習へ導く入口
   - 登場薬剤・薬効群への展開

4. **国試対策ページ**
   - 頻出薬リストと効率的学習パス
   - 各薬剤詳細への深掘りリンク

5. **臨床実践ページ**
   - 処方文化の5大法則
   - 理論と現実のギャップを埋める

#### クロスリファレンスの3原則
- **学習の深化**：リンクが理解を深める
- **文脈の拡張**：新たな視点を提供
- **実用的価値**：比較・選択に役立つ

#### 典型的な学習パス例
- 国試対策型：頻出薬→個別解説→薬効群→臨床応用
- ストーリー駆動型：感動→科学的理解→発展学習
- 臨床疑問解決型：実践的問い→エビデンス→処方文化
